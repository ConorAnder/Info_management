Rules applied to columns or tables to enforce data integrity.

* **PRIMARY KEY**: Uniquely identifies each row
* **FOREIGN KEY**: Enforces relationships between tables
* **UNIQUE**: No duplicate values allowed
* **NOT NULL**: Column must have a value
* **CHECK**: Column values must meet a condition

Constraints can be defined at two levels:
1. Column-level Constraints: Declared within a column definition.
2. Table-level Constraints: Declared separately after all columns.
